Данный проект реализован по микро-сервисной архитектуре. Авторизация вынесена в отдельный сервис. Покупка билета запускает событие в RabbitMQ, которое обрабатывает notification_service, осуществляя рассылку e-mail уведомлений. Сервисы обмениваются событиями асинхронно, что должно снижать связанность и повышать отказоустойчивость системы
